### Feminist Library
*Team of 4 · Mob programming*

Console application for managing a book catalogue (create, read, update, delete), built in plain Java with a layered MVC architecture and JDBC persistence.

`Java` `JDBC` `PostgreSQL` `JUnit` `Mockito`

- Built as a mob programming team: contributed across all layers — controller, repositories, model and database.
- Normalized relational model (books, authors, publishers, many-to-many genres) with ISBN validation and duplicate checks.
- Manual constructor-based dependency injection, without a framework.
- Designed the console interface: colour output and formatting for readability.

[Code](https://github.com/damcb1/FeministLibrary)

### Tarot Reader
*Pair project*

Web app that draws tarot cards from an external REST API and lets users save, edit and delete their readings.

`React` `JavaScript` `Axios` `REST APIs` `json-server`

- Built in a pair, splitting features and pairing on the harder ones: shared work across API integration, CRUD and UI.
- Consumed an external card API (images, names, meanings) and built full CRUD for saved readings against a json-server backend.
- Used PATCH for partial updates and reverse-chronological ordering of saved readings.

[Code](https://github.com/damcb1/tarot-project)
